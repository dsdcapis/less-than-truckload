# Less Than Truckload APIs

## Overview

This is the home of all artifacts created by the [Digital Standards Development Council (DSDC)](https://nmfta.org/digital-standards-development/) through the Digital LTL Council. The Digital LTL Council is comprised of industry leading Less-than-Truckload (LTL) transportation providers, logistics service providers, shippers, technology providers and organizations.

The purpose of the council is to facilitate collaboration, automation, standardization, and digitalization across all LTL industry participants with the hopes of elevating the industry together.

## Folder Structure

### api-scopes

The [api-scopes](./api-scopes) folder contains high-level scopes for APIs that will be created within this repository. Each scope document provides an overview of the API's purpose, functionality, and key features. These scopes serve as a reference for developers and stakeholders to understand the objectives and boundaries of each API project.

### api-prds

The [api-prds](./api-prds) folder contains Product Requirement Documents (PRDs) for APIs within this repository. Each PRD provides detailed requirements, use cases, and specifications that guide the development and implementation of the APIs.

### api-brds

The [api-brds](./api-brds) folder contains Business Requirements Documents (BRDs) for APIs within this repository. Each BRD provides detailed business requirements that guide the development and implementation of the APIs.

### api-specs

The [api-specs](./api-specs) folder contains OpenAPI specification files for the APIs in this repository. These specifications define the structure, request/response formats, and other technical details of each API in a standardized format.
